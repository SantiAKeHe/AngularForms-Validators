Kendo UI for Angular

Kendo UI for Angular is a commercial UI component library by Progress Telerik. It provides rich, accessible components — including dropdowns, date pickers, grids, and more — that integrate natively with Angular Reactive Forms via ControlValueAccessor. No adapters or wrappers needed.

Step 1 — Install the package

Add the Kendo Angular dropdowns package and its peer dependencies using npm. Kendo UI requires a license for production use. 

npm install --save @progress/kendo-angular-dropdowns @progress/kendo-angular-common @progress/kendo-theme-default

Step 2 — Import the theme in styles.scss

Kendo components rely on a CSS theme. Add the default theme to your global stylesheet. 

/* styles.scss */
@use '@progress/kendo-theme-default/dist/all';

Step 3 — Use standalone utility arrays

Kendo UI ships standalone utility arrays (e.g. KENDO_COMBOBOX, KENDO_DROPDOWNLIST) that contain all the components and directives needed for that widget. Spread them into the imports array of your standalone component — no NgModule required. 

import { KENDO_COMBOBOX } from '@progress/kendo-angular-dropdowns';
import { KENDO_DROPDOWNLIST } from '@progress/kendo-angular-dropdowns';

@Component({
  standalone: true,
  imports: [
    ReactiveFormsModule,
    ...KENDO_COMBOBOX,      // spread — includes ComboBoxComponent + directives
    ...KENDO_DROPDOWNLIST,  // spread — includes DropDownListComponent + directives
  ],
})

ComboBox — pick from a list or type a custom value

<kendo-combobox> is an autocomplete-style dropdown. The user can either select from the provided data list or type a custom value that is not in the list. Both actions update the bound FormControl.

How to wire it up

// 1. Define the form
form = this.fb.group({
  country: this.fb.control<string | null>(null, Validators.required),
});

// 2. Expose a typed getter
get country(): FormControl<string | null> {
  return this.form.controls.country;
}

// 3. Define the data array
countries: string[] = ['Ecuador', 'Colombia', 'Peru', 'Chile', 'Argentina'];
<!-- 4. Bind in the template -->
<form [formGroup]="form">
  <kendo-combobox
    formControlName="country"
    [data]="countries"
    placeholder="Select a country">
  </kendo-combobox>
  <app-error-message [control]="country" />
</form>

DropDownList — pick only from a predefined list

<kendo-dropdownlist> is a strict selection dropdown. The user must pick from the provided options — typing a custom value is not allowed. Use this when the value must be constrained to a known set.

How to wire it up

// 1. Define the form
form2 = this.fb.group({
  language: this.fb.control<string | null>(null, Validators.required),
});

// 2. Expose a typed getter
get language(): FormControl<string | null> {
  return this.form2.controls.language;
}

// 3. Define the data array
languages: string[] = ['TypeScript', 'JavaScript', 'Python', 'Java', 'Go'];
<!-- 4. Bind in the template -->
<form [formGroup]="form2">
  <kendo-dropdownlist
    formControlName="language"
    [data]="languages"
    [defaultItem]="'Select a language'">
  </kendo-dropdownlist>
  <app-error-message [control]="language" />
</form>

ComboBox vs DropDownList — side by side

Choose the right component based on whether your use case requires strict value constraints or allows user-typed custom input.

Feature ComboBox DropDownList
Custom values Yes No
Typing to filter Yes No
Predefined list only No Yes
Reactive Forms support Yes Yes
Import token KENDO_COMBOBOX KENDO_DROPDOWNLIST
HTML element <kendo-combobox> <kendo-dropdownlist>
