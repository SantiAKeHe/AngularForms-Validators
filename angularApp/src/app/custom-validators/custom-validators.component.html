Topic 03

Custom Validators

Three types of custom validators in Angular 19: simple, factory (with parameters), and cross-field. Each block shows the validator source code and a live demo form.

Simple Validator

noSpacesValidator

A single-responsibility validator: it checks whether the field value contains spaces and returns { noSpaces: true } when it does. Applied directly to a FormControl alongside built-in validators.

no-spaces.validator.ts

export function noSpacesValidator(): ValidatorFn {
  return (control: AbstractControl): ValidationErrors | null => {
    const hasSpaces = (control.value as string)?.includes(' ');
    return hasSpaces ? { noSpaces: true } : null;
  };
}

Apply to a FormControl

username: this.fb.control<string>('', [
  Validators.required,
  noSpacesValidator()
])

Factory Validator

minAgeValidator(minAge)

A validator factory — a function that accepts a parameter and returns a ValidatorFn. Use this pattern whenever the rule depends on a configurable value. Returns { minAge: { required, actual } } when validation fails.

min-age.validator.ts

export function minAgeValidator(minAge: number): ValidatorFn {
  return (control: AbstractControl): ValidationErrors | null => {
    const age = control.value as number;
    return age < minAge
      ? { minAge: { required: minAge, actual: age } }
      : null;
  };
}

Apply with a parameter

age: this.fb.control<number | null>(null, [
  Validators.required,
  minAgeValidator(18)
])

Cross-field Validator

passwordMatchValidator

A cross-field validator is applied at FormGroup level, not on a single control. It has access to all sibling controls and can compare them. Returns { passwordMismatch: true } when the two fields differ.

password-match.validator.ts

export function passwordMatchValidator(): ValidatorFn {
  return (group: AbstractControl): ValidationErrors | null => {
    const fg      = group as FormGroup;
    const pass    = fg.controls['password'].value;
    const confirm = fg.controls['confirmPassword'].value;
    return pass !== confirm ? { passwordMismatch: true } : null;
  };
}

Apply at FormGroup level

this.fb.group(
  {
    password:        this.fb.control<string>('', Validators.required),
    confirmPassword: this.fb.control<string>('', Validators.required),
  },
  { validators: passwordMatchValidator() }
)
